Example API Endpoints

Here are some example endpoints you can use once you have your API token. Please note the required regulations for each query parameter:

  • GET /api/dog-breeds?page=1&limit=10 - Retrieve a paginated list of dog breeds (requires `page` and `limit` query parameters). - The `page` parameter must be a positive integer, and `limit` must be an integer between 1 and 100.
  • GET /api/dog-breeds/{id} - Retrieve detailed information about a specific dog breed by its ID.
  • GET /api/dog-breeds/filter - Use multiple query parameters to filter the dog breeds by various characteristics. - Available filters:
    • Name: The common name of the breed (string).
    • Origin: The country or region of origin (string).
    • Type: The breed classification (e.g., "Sporting", "Terrier") (string).
    • Unique Feature: Distinctive trait (string).
    • Friendly Rating: An integer between 1 and 10 (inclusive).
    • Life Span: An integer (number of years).
    • Size: The typical size classification (small, medium, large, giant) (string).
    • Grooming Needs: Level of grooming required (string).
    • Exercise Requirements: Integer representing hours per day (integer).
    • Good with Children: Boolean indicating if breed is good with children (boolean).
    • Intelligence Rating: An integer between 1 and 10 (inclusive).
    • Shedding Level: Level of shedding (string, e.g., low, medium, high).
    • Health Issues Risk: Level of health issues (e.g., low, medium, high) (string).
    • Average Weight: Weight of the breed in kilograms (decimal or integer).
    • Training Difficulty: An integer between 1 and 10 (inclusive).
    - You can also add pagination parameters:
    • Page: A positive integer representing the page number (integer).
    • Limit: Number of items per page (integer between 1 and 100).
    - Example: `/api/dog-breeds/filter?friendly-rating=8&size=medium&lifespan=12&page=1&limit=10` - This will return breeds with a friendliness rating of 8, medium size, and lifespan of at least 12 years, showing the first 10 results on page 1.

All endpoints require a valid API token in the request header for authorization.
